When looking to start a new project or update an existing one, you might consider a web user interface framework like UIKit. In this article, we'll look at some of the powerful features and potential hazards of using a framework over custom JavaScript and CSS.

Although I specifically refer to UIKit these points are general enough that they apply to most UI frameworks including Bootstrap and Foundation.

Getting Started

The best place to start is at the beginning

UIKit

Pros: Working with a UI Framework provides a starting process with documentation for getting started, it makes the decision making and configuration portion of starting a new project easy.

Cons: A few more steps before starting to write code (downloading, finding a build process etc)

Custom Code

Pros: Vanilla HTML/CSS/JS are simple to get straight to writing code.

Cons: Adding preprocessors and transpiling (es6, coffeescript, etc) can drown a developer in choices making it hard to know the best place to start.

Winner? assuming you already have a proficiency in Custom HTML/CSS/JS it's probably easier to start with custom code if you haven't used a UI framework before.

Productivity

Writing code efficiently to create a product.

UIKit

Pros: Providing an extensive set of components makes it easy to stay productive without having to wait for design to prototype functionality and do user testing.

Cons: There are some coding practices that are unique to each UI Framework, so it does take some time to research how to use components in more complex interactions.

Custom Code

Pros: HTML/CSS/JS are the native languages of the web; any time spent practicing these skills will always be transferable to any other web project.

Cons: Since no components are really provided outside of standard HTML5 elements it takes quite a lot of time to build everything from scratch.

Winner? assuming you are building an application or product that needs the components provided by a UI framework, you probably find you are most productive with UIKit.

Maintenance

Maintaining, updating, and adding to an existing code base.

UIKit

Pros: Using a UI framework like UIKit gives the developer a standardized process for adding and editing components in an existing project.

Cons: Updating to new versions can bring breaking changes that require time and effort to research and address.

Custom Code

Pros: Without the UI framework dependencies you never have to worry about breaking updates. Also, it can sometimes be easier to create custom components in custom code than trying to build components into an existing framework.

Cons: Without a standard organization it can be tough to sometimes to return to an old project after not having worked on it for a time.

Winner? custom code gives you a lot more flexibility to change to new architectures and technologies, but if you're mostly doing content updates it would be easiest to update a UIKit project.

Collaboration

Working with other developers on the same code base.

UIKit

Pros: A standard organization and best practice recommendations that come with a UI framework can make collaborating on a project a bit easier since less code design decisions need to be discussed. Also, it can be easier for a new developer to get up to speed on an existing project if it leverages the practices recommended by the UI framework's documentation.

Cons: It can be tough to sometimes agree on a UI framework that everyone likes to use.

Custom Code

Pros: It can be easier to pivot in architecture decisions to more specifically address the needs of the product as it matures.

Cons: Without the standardization, the project can turn into a hot mess that is hard to read and update for new developers on the project.

Winner? the best practices and documentation really make UIKit a clear winner for collaboration.

Performance

Keeping a website or application lean and performant.

UIKit

Pros: UI frameworks like UI Kit tend to optimize components over time especially with outside contributions.

Cons: Frameworks tend to have components and functionality that isn't needed adding some amount of bloat to the assets that are downloaded by the user.

Custom Code

Pros: The developer can customize the UI to the exact needs of the product and keep it lean.

Cons: It can take a lot of time and energy to optimize custom code.

Winner? although you could spend time making a custom build of UIKit, your best chance for highly optimized code is to build it custom for your project.

Accessibility

Ensuring a product can be used by a large audience with diverse accessibility needs.

UIKit

Pros: Overtime, open source UI frameworks like UIKit tend to improve accessibility as developers file issues and submit pull requests. UIKit has a number of issues it has closed that has improved it's adherence to standards like Aria from W3C.

Cons: Not all projects are open to outside contributions and not all projects see the need to be strict in supporting accessibility standards. For example, as of writing this, this issue with the Dropdown has not been fully resolved.

Custom Code

Pros: No need to wait for issues to be fixed and updates released.

Cons: Working alone or in a small team on a custom code base can make it hard to stay abreast of accessibility needs and strategies.

Winner? UIKit has done a lot of the legwork for you and makes it easy to get great accessibility from the start.

Conclusion

Ultimately it comes down to the developer's preferences and the needs of the project, but for the vast majority of simple projects, a UI framework like UIKit provides so much out of the box that it's worth trying even if it's just for early prototypes.

If you are working on a project with a lot of resources or are planning to continually optimize and tweak it over a long period of time, you might find a custom code base make it possible to really dial in the performance and user experience that best suit your needs.
